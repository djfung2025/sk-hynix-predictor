GitHub Sync 配置（云端多端同步）

配置 GitHub 同步后，您的数据将被直接提交并保存在您的 repository 中，系统将自动触发 github.io 重新部署，确保无论在手机、iPad 还是其他电脑上，访问到的都是最新的统一数据库。

请确保该仓库已开启 GitHub Pages (github.io)
必须包含 repo 或 contents: write 权限。您的 Token 将仅被保存在您本地浏览器的 LocalStorage 中，不会上传给任何第三方。

如何部署到 GitHub Pages？

  1. 在 GitHub 上创建一个新的公开或私有仓库，例如命名为 sk-hynix-predictor。
  2. 将此项目的所有文件（index.html, styles.css, app.js, data.json）推送到该仓库。
  3. 在仓库的 Settings -> Pages 菜单中，将 Build and deployment 的 Source 设置为 Deploy from a branch，选择 main 分支的 / (root) 文件夹并保存。
  4. 几分钟后，您的网站将通过 https://<your-username>.github.io/<repo-name>/ 访问。
